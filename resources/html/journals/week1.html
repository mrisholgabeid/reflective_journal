The following is my reflective journal for Developing a Digital Identity. This is entry number one which covers the introduction of the module and what we will be learning overall.

In the introdcuction we covered the main/key features in writing HTML and CSS code, such as the syntax. We went through the syntax of both HTML and CSS so that we could be familiar in order to help us understand it better when it came to writing the code during the development of our own websites for the assignments. The main difference that we saw in the 2 languages is that HTML consists of elements, whereas CSS consists of rules.

HTML

A BRIEF HISTORY:

HTML was introduced in 1989 by Tim Berners-Lee of CERN. Since its introduction and throughout its time, it has seen many updates and changes from its very first version HTML 1.0 going to HTML 2, HTML 3.2, HTML 4, XHTML 1, XHTML 2, HTML 5 and finally HTML 5.1. All of the changes and updates took effect within 7 years of the introduction of HTML... For more information please see W3.org.

SYNTAX:

Despite the many changes and updates that HTML has gone through, its syntax has always remained constant which made it easier for people to adopt to the new changes. When taking about HTML syntax we are refering to how HTML is structured in order for the browser to be able to understand we as developers need it to output.

HTML syntax is basically divided into 2 main parts that any web developer, whether front-end or back-end should be familiar with. The 2 main parts are:

  • Format/Structure
  • Tags
FORMAT:

The format of writing HTML code is very basic and includes a few rules that are easy to understand. The rules are as follows:

  • Always start with an opening HTML tag <html>, followed by an opening HEAD tag which is, <head>, then a title tag which is, <title> in which you can writer the title of your webpage such as MY FIRST WEBPAGE followed by a closing title tag which is, </title> and a closing head tag which is, </head>. After that you introduce the opening body tag which is, <body> followed by everything that will be included in the webpage when it is opened, for example: a heading 1 title; <h1> WHY I CHOOSE HTML? </h1>, a paragraph; <p> I love HTML because it is the easiest, and most importantly, everyone should know it if they want to develop a website </p> and an image; <img src="img/html.png">, then followed by the closing body tag which is, </body> and finally the closing html tag which is, </html>. See example 1.1 below... For more examples and explanations of HTML syntax please see W3schools.com.
    • Example:
    <img src="img/html.png">
    Description:
    When the browser reads the <img> tag above, it will understand that we need it to display an image on the webpage. The syntax behind it is that, it will read the img tag and refer to the src or source attribute, which will tell it the exact location of the image that we want to display. In this case, the image is located in an img folder, and using the /, it knows to go in that folder where it will find the image called html with a file type .png. With this code, the browser will know exactly where to go in order to find the image and display it on the webpage.
  • Should always begin with an opening and closing tag of the same element.
    • This means that for every tag that you open, must be closed. Except from a few exceptions such as the image tag <img>, every other tag that is opened should have a corresponding closing tag. For example: The HTML tag:<html> has a corresponding closing tag of </html>
CODE EXAMPLE: 1.1
<html>

<head>

<title> MY FIRST WEBPAGE </title>

</head>

<body>

<h1> WHY I CHOOSE HTML? </h1>

<p> I love HTML because it is the easiest, and most importantly, everyone should know it if they want to develop a website</p>

<img src="img/html.png">

</body>

</html>
OUTPUT EXAMPLE: 1.1

WHY I CHOOSE HTML?

I love HTML because it is the easiest, and most importantly, everyone should know it if they want to develop a website

Important:

In order for a web browser to understand that the document you are trying to open is a website/webpage, you have to make sure that you save the document containing all of the HTML code for your website with a .html or .htm extension after the name of your document. Without either one of these extensions, the web browser will not understand that the document you are trying to view is a webpage/website hence it will fail to launch/open it.
TAGS:

  • Tags
    • HTML tags follow a certain writing convention which differs depending on whether it is an opening tag or closing tag. As long as you are writing an HTML tag, it much consist of a < in the front of the tag, the name of the tag; such as html and a > at the end of the tag. All together, a HTML tag reads <html> which is known as an opening tag. The closing tag follows the same rules but it includes an extra element which is / sign which is used just before writing the name of the tag itself, such as </html>. With the opening tag and closing tag in place, the browser will be able to differentiate where the starting and ending points of the website and/or elements are.

CSS

A BRIEF HISTORY:

CSS was introduced in November 4th, 1994 by Hakon Wium of CERN. Similar HTML, CSS has experienced its fair share of updates and upgrades which made it more and more powerful after every update that it went through. There have been a 3 major versions of CSS which are CSS 1, which was used by the W3C - World Wide Web Consortium who introduced the use of typography in the December, 1996, going to CSS 2 and finally the latest version which is CSS 3. As newer versions of CSS were introduced, more and more features were established making CSS a powerful tool when styling content on a webpage. In order to learn more about the history of CSS and what things you can do with it as a website developer, please see W3.org.

SYNTAX:

Even though CSS has gone through a few updates and upgrades throughout it's time, its syntax has always remained constant no matter the version. When we are refering to CSS syntax we are talking about how we can write CSS code so that our browsers can understand it and be able to differentiate it from other languages such as HTML or Java Script.

CSS syntax consists of rules which are divided into 2 main parts that every web developer should be familiar with. These are:

  • Selector
  • Declaration
SELECTOR:

A selector is simply an HTML element that we want to style on the webpage. A selector could be the body, an image, a video, a heading, an audio file and so no... or even html itself

Unlike HTML, declaring rules in CSS is a little bit different. Instead of using tags we simply write the name of the element that we want to style and then provide the declarations that will affect that element in the HTML which will be displayed in the webpage.

Example:
p {
   color:red;
}
Description:
The CSS line above is targeting the p or paragraph tag in the HTML. This is a generalized tag because it is targeting all paragraph tags in the HTML which will be displayed on the webpage. The syntax behind the code is that, the selector, in this case the paragraph tag will be affected by the declarations specified. The declarations will be discussed further later. For an example, see the following example below:
HTML CODE:
<p> I am a red paragraph </p>
CSS CODE:
p {
   color:red;
}
OUTPUT:

I am a red paragraph
